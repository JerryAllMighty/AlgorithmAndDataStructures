상태 정의 → 점화식 도출 → 기저 사례 설정 → 구현 방식 선택 (Top-down vs Bottom-up).

① 상태 정의 (State Definition)가장 중요   
$dp[i]$나 $dp[i][j]$가 무엇을 의미하는지 문장으로 정의.   
예: $dp[i]$ = $i$원를 만드는 데 필요한 동전의 최소 개수.
예: $dp[i][j]$ = $i$행 $j$열까지 도착했을 때 얻을 수 있는 최대 점수.

② 점화식 찾기 (Relation)
"현재 상태 $dp[i]$를 구하기 위해 바로 직전 단계($i-1, i-2$ 등)에서 어떤 값을 가져와야 하는가?"를 고민합니다.
핵심 질문: $i$번째 단계에 도달하기 전의 '마지막 선택'은 무엇이었나?
예: 계단 오르기라면 "1칸 전에서 왔나, 2칸 전에서 왔나?"
$\rightarrow$ $dp[i] = max(dp[i-1], dp[i-2]) + cost[i]

③ 기저 사례 설정 (Base Case)
더 이상 쪼갤 수 없는 가장 작은 문제의 답을 미리 정합니다.
예: $dp[0] = 0$, $dp[1] = 1$ 등.
IndexError 조심

④ 구현 방식 선택
Top-Down (재귀 + Memoization): 큰 문제부터 호출.
가독성이 좋고 점화식을 코드로 옮기기 편함.
Bottom-Up (반복문 + Tabulation): 작은 문제부터 채워 나감. 
실행 속도가 빠르고 스택 오버플로우 걱정이 없음.

Memoization (Top-Down): 
재귀(Recursion) 기반. 필요한 결과만 계산하며, 이미 계산된 결과는 메모리(캐시)에 저장해 둡니다.
Tabulation (Bottom-Up): 
반복문(Iteration) 기반. 작은 문제부터 시작하여 테이블(DP 배열)을 채워 나갑니다.
핵심 차이: 메모이제이션은 '필요할 때' 구하고, 타뷸레이션은 '미리 전부' 구해둡니다.

⑤ 최적화 (Space Optimization)
가끔 $dp[i]$를 구할 때 $dp[i-1]$만 필요하다면, 배열 전체가 아닌 변수 몇 개만으로도 해결 가능합니다.
